(Pipes):- Pipes in Nestjs are functions that transform a validate data before it reaches a controller method.

Pipes are used to clean, sanitize, or convert data to a specific format, ensuring that the data received by the controller is in a suitable state for further processing.

Pipes are used for:- 

(i). Data Validation (ensuring inputs match expected rules such as data types, length, formats. and this help to prevent errors and improves data quality)(class-validator).
(ii). Data Transformation (changing inputs into desired formats. such as parse string JSON, convert data types, format dates )(class-transformer).
(iii). Data Sanitization (cleaning inputs to remove dangerous or unwanted content such as HTML tags, malicious scripts. this helps to protect your application from security vulnerabilities )(class-sanitizer).

                              DTOs are often used to decouple the different layers of the application and to make the code more readable and maintainable.

Dependency Injection:- Dependency Injection is a software design pattern that involves passing dependencies( object or services) into an object rather than the object creating them itself. 
                       this promotes loose coupling, making the code more modular, testable, and maintainable.
            
            why Use Dependency Injection in NestJS  
            (i). Modularity:- DI encourages breaking down applications into smaller, reusable modules, improving code organization and maintainability. 
            (ii). Testability:- By injecting dependencies , you can easily mock or stub them during testing, isolating the component under test and simplifying testing procedures.
            (iii). Reusability:- DI allows you to reuse components across different parts of your application, promoting code efficiency and reducing redundancy( repeat same code).
            (iv). Maintainability:- DI make it easier to modify and extend your application without affecting existing code.
            (v). Configurability :- DI enables you to configure dependencies dynamically, allowing for customization and flexibility.

( Types of Dependency Injection):- 

    1. Intra-modular Dependency Injection:- Intra-module Dependency Injection refers to injecting one provider (class/service) into another within the same module.
        ( Example:- inject user Service to the user controller )

    2. Inter-modular Dependency Injection:- Inter-module Dependency injection is when a provider(class/service ) from one module is injected into another module using exports and imports.
        ( An inter-modular dependency occurs when a component in one module (depends on)( injected into) a component in another module).
        ( Example:- user and tweet models)

    3. Circular Dependency Injection:- when two or more modules or components have a direct or indirect depencdency on each other, forming a cycle.
        ( Problem: This can cause runtime errors or undefined injections unless handled carefully. )
        ( forwardRef tells NestJS to resolve the dependency later, breaking the circular chain)
        ( Ex:- auth and user) ( both are passing their sevices)

Question:- assume have columns and new model have code that have 2 more columns and  1 column removed that have a data , now explain both synchronize true/false scnerio on that and  what effects also on data. 

ChatGPT said:
Perfect — now let’s go deep into your case with a clear scenario using synchronize: true vs false.

🎯 Situation:
You already have a database table User with these columns:

ts
Copy
Edit
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @Column()
  email: string;
}
And your database already has this data:

id	name	email
1	Alice	alice@example.com
2	Bob	bob@example.com

Now, you change your model in code like this:

ts
Copy
Edit
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @Column()
  age: number;             // ✅ New column

  @Column()
  isActive: boolean;       // ✅ New column

  // ❌ Removed the "email" column
}
🔁 Now see both scenarios:
✅ Scenario 1: synchronize: true
NestJS/TypeORM sees your updated model and says:

"Oh! There are 2 new columns and 1 old column is gone. Let me fix the database automatically."

What happens:
Adds age and isActive columns → ✅ OK

Drops email column → ❌ Deletes that column and all its data

Now your table looks like:

id	name	age	isActive
1	Alice	NULL	NULL
2	Bob	NULL	NULL

🔴 All email data is lost forever, because synchronize blindly dropped the column.

🚫 Scenario 2: synchronize: false
TypeORM does nothing to the table.

What happens:
age and isActive are not created

email is not deleted

Database stays exactly like this:

id	name	email
1	Alice	alice@example.com
2	Bob	bob@example.com

But now when your application tries to save a user, or fetch age/isActive, you might get:

Errors like “column doesn’t exist”

Inconsistencies between code and database

✅ Good thing: No data is lost
❌ Bad thing: You need to manually create migrations or SQL to update schema.

🧠 Summary:
Action	               synchronize: true	             synchronize: false
Adds new columns	   ✅ Automatically	               ❌ You do it manually
Removes old columns	   ✅ Automatically (dangerous) 	   ❌ You do it manually
Deletes column data	   ❌ YES (irreversible loss)	   ✅ NO data loss
Best for	           Development                   	 Production

📌 Best Practice:
(i). Use synchronize: true in local dev only when you're okay losing data.

(ii). Use synchronize: false + migrations in production to:
      --Keep control
      --Protect data
      --Review every change

( SYNCHRONOUS CONNECTION):- 
    -- synchronous connection establishes a direct, immediate link between your application and the database.
    -- when your code requests data, the execution halts until the database responds.
    -- this blocking behaviour can impact application performance, especially with time-consuming queries.

( ASYNCHRONOUS CONNECTION):- 
    -- asynchronous connection utilize promises or async/await.
    -- your code can continue executing other tasks while the database query is running in the background.
    -- this improves application reponsiveness and prevents blocking.

                   Allows recovery of accidentally deleted data.

Decorators are a powerful syntactic feature in Typescript that allows you to add functionality or modify the behaviour of a class, method, property or a method parameter at runtime.
Decorators are essentially fucntions that are applied using @ Symbol before the declaration you want to declare.
 
//----------( SET METADATA DECORATOR)

the @SetMetadata(key,value) decorator is a build-in Nestjs decorator that allows you to associate arbitrary metadata with a specific handler or controller.
key: This is a string that serves as the identifier for your metadata.
value: This can be any js value - string, boolean, object, array.
You can apply @SetMetadata directly to a route handler ( within a controller method) or to an entire controller.


the Reflector is a helper class used to access metadata that has been set using decorators like @SetMetadata. 
It is particularly useful in custom guards, interceptors, roles access and other places where you want to read metadata at runtime.
